# SUPER ADMIN (DEVELOPER) ROLE

Create a fourth role called:

```text
superadmin
```

Role hierarchy:

```text
Super Admin
    ↓
Admin
    ↓
Faculty
    ↓
Student
```

The Super Admin is the highest authority in the system and has complete control over the platform.

Super Admin accounts must NOT be publicly creatable.

Super Admin accounts can only be created directly in the database.

There should be no public signup page for Super Admins.

---

# ADMIN MANAGEMENT

Only Super Admin can manage Admin accounts.

Features:

Create Admin

Fields:

* Full Name
* Email
* Mobile Number
* Password

Edit Admin

Block Admin

Unblock Admin

Delete Admin

Reset Admin Password

Activate Admin

Deactivate Admin

View Admin Activity

Admin accounts must not be manageable by other Admins.

# ROLE-BASED ACCESS CONTROL

Roles:

```text
superadmin
admin
faculty
student
```

Permissions:

Super Admin:

* Full Access

Admin:

* No Access To Super Admin Pages

Faculty:

* No Access To Admin Pages
* No Access To Super Admin Pages

Student:

* Student Pages Only

Protect all routes and database queries according to role.

Never rely only on frontend role checks.

Validate permissions at database and API level.
